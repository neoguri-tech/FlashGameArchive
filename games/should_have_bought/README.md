# Should Have Bought

최근 일봉 구간을 보고 매일 매수, 매도, 관망을 선택하는 트레이딩 리플레이 게임입니다.

## 실행

저장소 루트에서 Live Server를 켠 뒤 아래 주소를 엽니다.

```text
http://127.0.0.1:5500/games/should_have_bought/index.html
```

## 데이터

로비에서 선택한 종목의 저장 일봉 스냅샷을 우선 사용합니다. 각 파일은 Yahoo Finance 차트 응답을 개발 시점에 한 번 받아 정적 JSON으로 저장한 것입니다.

현재 저장 파일은 다음과 같습니다.

- `data/kospi-ks11.json`: KOSPI
- `data/btc-krw.json`: Bitcoin KRW
- `data/eth-krw.json`: Ethereum KRW
- `data/sol-usd.json`: Solana USD
- `data/soxl.json`: SOXL

저장 데이터 파일을 불러오지 못하면 원격 Yahoo Finance 차트 API를 한 번 더 시도합니다. 정적 사이트에서는 CORS나 서비스 제한으로 실패할 수 있으므로, 원격 요청도 실패하면 랜덤 생성 일봉 데이터로 자동 전환합니다.

## 현재 룰

- 로비에서 투자 대상을 선택합니다.
- 초기 평가금액은 원화 종목 10,000,000원, 달러 종목 $10,000입니다.
- 시작 시 예열 10번째 봉의 종가 기준으로 초기 평가금액의 절반은 현금, 절반은 현물로 지급됩니다.
- 한 판은 예열 10개봉을 먼저 보여준 뒤 30거래일을 플레이합니다.
- 매수는 입력한 거래 수량만큼 선택 종목 단위를 삽니다.
- 매도는 입력한 거래 수량만큼 보유 단위를 팝니다.
- 비싼 종목도 거래할 수 있도록 거래 수량은 소수 단위를 허용합니다.
- 입력 수량이 0이거나 매수/매도 가능 수량보다 크면 해당 거래 버튼이 비활성화됩니다.
- 매수, 매도, 관망은 마지막으로 보이는 종가에서 처리되고 다음 일봉이 흔들리며 공개됩니다.
- 차트에는 현재가 점선과 보유 포지션의 평단가 점선이 함께 표시됩니다.
- 다음 일봉 공개 시간은 3-6초 사이에서 캔들 변동성이 클수록 길어집니다.
- 마지막 날 평가금액이 점수이며, 최고 기록은 브라우저 로컬 저장소에 저장합니다.
- 결과창에는 다음 종가를 모두 알고 풀매수/풀매도했을 때의 이론상 최대 수익률도 함께 표시합니다.
- 현재 수익률에 따라 개미 투자자 아바타가 깡통, 돔황챠, 본전 기도, 가즈아, 대호황, 텐베거 등 10단계로 바뀝니다.

## 향후 계획

- 현물 모드가 안정화되면 롱/숏, 레버리지, 청산을 포함한 선물 모드를 추가할 예정입니다.
